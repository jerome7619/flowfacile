Série Formulaires — Partie 5/5 : Rapport hebdomadaire des demandes en attente ou non traitées

Juin 2025 · 10 min · Intermédiaire · Pas à pas

Chaque vendredi matin, ce flow génère automatiquement un tableau récapitulatif de toutes les demandes encore en attente et l'envoie par courriel — sans aucune intervention manuelle.

⏱ Temps de réalisation estimé : 20 à 30 minutes — le plus court des trois flows de la Partie 3.
Étape 1
Déclencheur : vendredi matin à 10h

Créez un nouveau flow avec le déclencheur Planification (Récurrence) :

Étape 2
Récupérer les demandes en attente depuis SharePoint

Ajoutez « Obtenir des éléments » (SharePoint) avec ce filtre et ce tri :

ListeDemandesFormation
FiltreStatut eq 'En attente d''approbation' or Statut eq 'Transmise'
Trier parDateFormation croissant
Étape 3
Sélectionner les colonnes à afficher dans le rapport

Ajoutez l'action « Sélectionner » pour choisir quelles colonnes apparaîtront dans le tableau du rapport :

Debody/value (résultat Obtenir des éléments)dynamique
Mapper : Demandeuritem()?['Demandeur']
Mapper : Formationitem()?['Formation']
Mapper : Date de la formationitem()?['DateFormation']
Mapper : Gestionnaireitem()?['Gestionnaire']
Mapper : Date de soumissionformatDateTime(item()?['DateSoumission'],'yyyy-MM-dd HH:mm')
💡 Pourquoi l'action Sélectionner? Elle permet de choisir exactement quelles colonnes apparaissent dans le tableau — et de renommer les en-têtes en français lisible plutôt qu'avec les noms techniques SharePoint.
Étape 4
Générer le tableau HTML

Ajoutez l'action « Créer un tableau HTML » — elle transforme automatiquement les données en un tableau HTML formaté :

DeRésultat de l'action Sélectionnerdynamique
ColonnesAutomatique
Étape 5
Vérifier que le tableau contient des données

Un tableau HTML vide généré par Power Automate contient toujours les balises d'en-tête — environ 101 caractères. Ajoutez un Compose pour calculer la longueur, puis une Condition :

Composelength(body('Créer_un_tableau_HTML'))expression
Condition — ValeurRésultat du Composedynamique
Opérateurest supérieur à
Valeur101
⚠️ Pourquoi 101? Un tableau HTML sans données contient quand même les balises <table>, <thead>, <tr>, <th> — ce qui représente environ 101 caractères. Si la longueur dépasse 101, il y a de vraies données à envoyer. Cette astuce évite d'envoyer un courriel avec un tableau vide chaque vendredi.
Étape 6
Envoyer le rapport par courriel (si données présentes)

Dans la branche « Si oui », ajoutez « Envoyer un e-mail (V2) » avec le tableau HTML dans le corps :

Àgestionnaire@formation-demo.ca
Objet📋 Rapport hebdomadaire — Demandes de formation en attente
CorpsEnvoi automatisé chaque vendredi matin.

@{body('Créer_un_tableau_HTML')}
Est-ce HTMLOui
💡 Est-ce HTML : Oui est important — sans cette option, le tableau s'affiche comme du texte brut dans le courriel plutôt qu'un vrai tableau formaté.

← Partie 4 — Relance

Every Friday morning, this flow automatically generates a summary table of all pending requests and sends it by email.

Steps 1–6
Friday trigger → Get items → Select columns → Create HTML table → Check length > 101 → Send email

The key trick: check length(HTML table) > 101 before sending — an empty HTML table still contains ~101 characters of header markup. Set Is HTML: Yes on the email to render the table properly.
